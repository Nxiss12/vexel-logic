## üíª Development Environment

### Recommended Setup
**Best:** Run your own Linux distribution
- Any distro: Debian, Ubuntu, Mint, Fedora, etc.
- Windows users: Use VirtualBox or dual-boot
- Mac users: Install VirtualBox/UTM

**Alternative:** Use Omnissa Horizon (School's Linux Virtual Desktop)

### Required Software
- **Text Editor:** gedit, vim, emacs, VS Code, etc.
- **Compiler:** GCC (GNU Compiler Collection)
- **Standard:** ANSI C (C89) / ISO C (C90)

### Standard Compilation Command
```bash
gcc -Wall -ansi -pedantic-errors -o program_name source.c
```

**Flags Explained:**
- `-Wall`: Show all warnings
- `-ansi`: Use ANSI C standard
- `-pedantic-errors`: Enforce strict standard compliance
- `-o program_name`: Specify output executable name

## üìù Study Tips

### Daily Practice
1. **Code Every Day** - Even 30 minutes helps build muscle memory
2. **Type, Don't Copy** - Typing code helps you understand it better
3. **Debug Your Own Code** - Learn to find and fix errors independently
4. **Comment Your Code** - Practice explaining what your code does

### Understanding Concepts
1. **Draw It Out** - Visualize memory, pointers, data structures
2. **Explain To Others** - Teaching solidifies understanding
3. **Break It Down** - Complex problems into smaller pieces
4. **Ask "Why?"** - Don't just memorize, understand the reasoning

### Exam Preparation
1. **Practice Writing Code By Hand** - Exam may be written
2. **Understand Error Messages** - Know what they mean
3. **Review Your Coursework** - Understand feedback
4. **Time Yourself** - Practice under time pressure
